# Lite-1.5 Structural Channel Audit — Diagnostic Findings

**Date:** 2026-03-02
**Severity:** High (structural channel is noise for Llama models)
**Status:** Diagnosed, design direction decided, implementation post-launch
**Prerequisite:** `findings/2026-03-02-llama-neutral-50-bias.md` (lite-1.4 diagnosis)

## Summary

The lite-1.5 two-dimension scoring system (editorial + structural) deployed correctly and
the pipeline engineering works. However, the first 227 evaluations reveal that Llama models
produce near-constant structural scores — 86% of values fall on just two integers (55 and 58).
The structural channel does not differentiate content. SETL computed from these scores is an
artifact of the structural positive bias, not genuine editorial-structural tension.

**Root cause:** The anti-50 instruction ("reserve 50 for content with literally zero UDHR
connection") displaced the attractor from 50 to 55/58 rather than teaching discrimination.
The structural rubric's explicit UDHR article mappings anchor Llama to "this is tech content →
Art. 27 → always 58." This is Goodhart's Law applied to prompt engineering.

### SETL Sign Distribution

| Metric | Value |
|--------|-------|
| SETL positive (editorial > structural) | 25 (11%) |
| SETL negative (structural > editorial) | **200 (88%)** |

88% negative SETL = structural systematically exceeds editorial. This is not genuine

through the SETL formula.

### Content-Type Inconsistency → Weighted Mean Noise

Same story, same E and S scores, different weighted_means:

| hn_id | llama-3.3-70b WM | llama-4-scout WM | Δ | Cause |
|-------|-------------------|-------------------|------|-------|
| 47181542 | 0.112 | 0.064 | 0.048 | Different content_type classification |
| 47206009 | 0.112 | 0.080 | 0.032 | Different content_type classification |
| 47217812 | 0.080 | 0.112 | 0.032 | Reversed! |

Content-type is an LLM classification task. When models disagree on content_type for the
same story, CONTENT_TYPE_WEIGHTS produces different blend ratios, injecting noise into the
weighted_mean from identical E+S inputs.

## What Still Holds

- Pipeline engineering is correct: data flows, columns populate, math is right
- eval_history structured columns work: longitudinal queries are possible
- Two-dimension schema is architecturally sound: the measurement model is right
- Editorial dimension unchanged from lite-1.4: no regression (65% zero rate, same as before)
- The problem is Llama's scoring behavior on abstract/nuanced tasks, not the architecture

## Diagnosis

Llama's failure mode is not "can't see implicit rights." It's **"defaults to a safe
attractor on any ambiguous scoring task."** The anti-50 instruction didn't teach
discrimination — it told Llama where NOT to pile up, so it piled up somewhere else.
This is a fundamental limitation of Llama's capability on 0-100 integer scoring for
abstract constructs.

Llama CAN do:
- Binary classification (yes/no)
- Counting (how many sources linked?)
- Pattern matching (is there a byline?)
- Category assignment (content type — though with inter-model inconsistency)

Llama CANNOT do:
- Nuanced 0-100 scoring on abstract constructs
- Distinguishing "mildly relevant" from "moderately relevant" implicit UDHR alignment

## Design Direction: Model-Tiered Lite Architecture with TQ

### Decision

Replace the structural channel for Llama with **TQ (Transparency Quotient)** — binary
and countable verifiability indicators. Keep structural for Haiku (which may differentiate
at scale; needs cross-validation to confirm).

### Rationale (8-order knock-on analysis)

All 8 orders favored "design now, build post-launch":
- Structural noise is invisible to users (internal rater_evals, not page-level display)
- TQ is an untested construct — shipping it without pipeline validation risks errors
- Post-launch TQ benefits from real user feedback on what indicators matter
- No irreversible debt in either direction
- Velocity feedback loop is appropriate for first launch; quality loop post-launch

### Architecture Sketch

```
  LLAMA (Workers AI, free)              HAIKU (standalone, paid)
  ┌───────────────────────────┐        ┌──────────────────────────┐
  │ editorial: 0-100          │        │ editorial: 0-100         │
  │ tq_author: bool           │        │ structural: 0-100        │
  │ tq_sources_cited: int     │        │ (nuanced UDHR alignment) │
  │ tq_corrections: bool      │        │                          │
  │ tq_conflicts: bool        │        │                          │
  │ tq_funding: bool          │        │                          │
  │ → tq_composite: 0.0-1.0   │        │                          │
  └───────────────────────────┘        └──────────────────────────┘

  Storage: tq_* in own columns         Storage: hcb_structural_mean
  No cross-model consensus on           Consensus with full evals
  structural channel                     on structural channel

  SHARED: editorial → consensus works across all models
  SEPARATE: TQ (objective) vs structural (evaluative)
```

### Validation Plan

1. **Dry-run test** (this session): TQ prompt with Haiku via standalone evaluator —
   verify binary indicators are reliable. **DONE — see results below.**
2. **Llama TQ validation** (post-launch): Run TQ prompt through Workers AI pipeline
   to verify Llama produces reliable binary indicators (not just Haiku)
3. **Haiku structural cross-validation** (post-launch): Run Haiku on structural prompt
   for ~50 stories to determine if structural works for capable models
4. **TQ cross-validation** (post-launch): Compare LLM-generated TQ against automated
   HTML extraction (count `<a>` tags, detect byline patterns) for ground truth
5. **TQ schema implementation** (post-launch): New columns, model-specific prompt
   routing, consumer worker changes

### TQ Dry-Run Results (Haiku, 2026-03-02)

Tested via `scripts/tq-dryrun.mjs` against 4 real URLs with known transparency profiles.

| URL | TQ | Author | Sources | Date | Corrections | Conflicts | Methodology |
|-----|-----|--------|---------|------|-------------|-----------|-------------|
| github.com/anthropics/claude-code | 0.00 | false | 0 | false | false | false | false |
| cnbc.com (Paul Allen obit) | 0.15 | false* | 0 | true | false | false | false |
| byran.ee (open-source laptop) | 0.60 | false | 23 | true | false | false | true |
| anthropic.com (Dario statement) | 0.80 | true | 5 | true | false | false | true |

*CNBC author likely false negative due to 30k char content truncation (model noted this).

**Key findings:**
- **Full range used:** 0.00, 0.15, 0.60, 0.80 — 4 distinct scores across 4 URLs
- **Composite math correct:** Model applied the weighted formula accurately in all 4 cases
- **Verifiable specifics:** Author names ("Dario Amodei"), source counts with named examples
  ("RK3588 Brief Datasheet", "DoD AI Strategy document"), specific dates
- **Honest uncertainty:** Model noted content truncation limitation for CNBC; noted Anthropic's
  missing conflict disclosure "despite obvious business stakes"
- **Compare structural:** 4 real values vs 86% clustering on 2 integers

**Caveats:**
- Haiku results, not Llama. Llama may cluster on binary outputs differently.
- n=4 is too small for statistical claims. These validate the prompt design, not the construct.
- Content truncation (30k chars) may cause false negatives for indicators in article body.
